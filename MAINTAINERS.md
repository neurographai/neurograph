## How to Become a Maintainer

1. Make significant, sustained contributions to the project
2. Demonstrate understanding of the codebase and project goals
3. Be nominated by an existing maintainer
4. Receive approval from the project lead

## Responsibilities

- Review and merge pull requests
- Triage issues and respond to community questions
- Participate in architectural decisions
- Ensure code quality and test coverage
- Help onboard new contributors
