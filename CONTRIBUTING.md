# Contributing to NeuroGraph

First off, thank you for considering contributing to NeuroGraph! 🚀

NeuroGraph is an ambitious project integrating Rust and WebAssembly, and we welcome contributions ranging from bug fixes and documentation improvements to new features.

## Getting Started

1. Fork the repository on GitHub.
2. Clone your fork locally.
3. Set up the development environment by following the `DEVELOPING.md` guide.
4. Create a new branch for your feature or bugfix.

## Development Workflow

- Code primarily lives in the `crates/` directory (Rust) and `dashboard/` directory (React/TypeScript).
- Make sure to run the test suite and verify everything passes.
- Write tests for your new features.

## Pull Requests

1. **Keep it focused**: Submit one PR per feature or bug fix.
2. **Describe your changes**: Update the PR template with clear context and rationale.
3. **Pass checks**: Ensure formatting (`cargo fmt`, `npm run lint`) and tests (`cargo test`) pass.

## Community

Join discussions and ask questions on our GitHub Discussions or Discord!
